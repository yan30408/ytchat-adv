### 設置の手順 #######################################################################################
 1. config.cgi.defaultのファイル名をconfig.cgiに変更する（.defaultを消す）

 2. config.cgiを開いて、管理用パスなどを設定する
 
 3. サーバーによっては、index.cgiの一行目を書き換える。
    (#!/usr/bin/perl から #!/usr/local/bin/perl など。)

 4. ゆとチャのファイルをすべてサーバーにアップする

 5. index.cgiのパーミッション（属性）を<705>または<755>に変更する
   （サーバーによっては<700>などの可能性もアリ）

 6. ブラウザからチャットのTOPページ(http://○○○/ytchat-adv/)へアクセスして
    問題なく表示されれば設置完了

### ファイル構成 #####################################################################################
  <>内は一般的なパーミッション設定です。基本的に、「index.cgi」以外は設定する必要はありません。

 /ytchat-adv <705> このディレクトリ（フォルダ）名は変更可
  |
  |- index.cgi          <705> 基本CGI。パーミッション（属性）設定必須。
  |- config.cgi.default <604> 設定ファイル。ファイル名を「必ず」 config.cgi に変更してください
  |
  |- /lib <705> ライブラリ格納ディレクトリ
  |  |
  |  |- /pl
  |  |  |
  |  |  |- config-default.pl <604> デフォルト設定格納ファイル
  |  |  |- *****.pl          <604> その他処理ライブラリ
  |  |
  |  |- /js
  |  |- /css
  |  |- /html
  |
  |- /room   <705> 部屋データ格納ディレクトリ
  |- /logs   <705> 過去ログデータ格納ディレクトリ
  |- /rtable <705> ランダムテーブルファイル格納ディレクトリ
  |- /module <705> 同梱Perlモジュール格納ディレクトリ

## 不具合修正
以下の不具合修正
- 中括弧内に特定の文字列を含む発言ができない（JS正規表現のエラー）
- タブを新規タブグループへ移動させようとした際に処理が止まる
- ヘルプに水平線記法に関する記述が無かった
- 発言者カラーのランダム割り当て時に余計な `#` が含まれる
- ログの HTML で `color: undefined` などの記述が発生し得る
- simple モードだとログが２回出力される
- 秘話の発言者名を変更すると宛先が（視覚的に）消失する不具合
- ルーム内において、ルビ記法の変換結果が HTML として破綻していた
- BGM履歴にスクロールバーが出ない
- ローカルに保存したログで、アンカーが有効でない
- BCDice使用設定であっても、ゆとシートからのユニット作成時にゆとチャadv.向けのチャットパレットを取得してしまう
- ルームとログで文字装飾の結果が違うことがある（文字置換の処理順を合わせる）
- 背景画像変更時に表示される画像をクリックしても拡大表示されない
- キャラシートのJSONおよびチャットパレット読み込み時、一部サーバーのWAFに弾かれることがある
- 「後で公開」チェックがONの秘話とOFFの秘話を続けて発言したとき、表示上においてその区別がされないことがある
- 発言があっても入室メンバー一覧（および秘話宛先）が更新されない
- メモ欄で改行や`<`、`>`を入力して送信した際、入力欄の中でエスケープされたものに置き換わってしまう
- チャットパレットをクリックした際、クリック箇所が何らかのアイコン（[魔][刃][打]などの）上だと反応しない
- リスト記法の直後にテーブル記法があった場合に正しくパースできない不具合
- ログ閲覧時、dialog表示中は後ろをスクロールしないようにする
- ランダム表に関する不備の修正・改善
- すべての共有メモを削除しても初期状態と同様の表示にならない
- 先頭に空行がある共有メモを正しく追加できない
- ユニットのステータス名に括弧が含まれる場合に正常に表示・更新されない
- コンフィグ内の文字列置換定義で置換前の文字列があらわす正規表現内の文字クラスに、 Unicode 16 進において 4 桁以内に収まらない文字が含まれていた場合に、ルーム内で正しく置換されない
- ユーザー定義テキスト置換内の `&lt;`, `&gt;` がそれぞれ一度しか置換されない
- ログ画面における表組み記法が破綻することがある
- エラーメッセージに改行が含まれる場合に JSON の構造が壊れる
- BCDice-API の v1 を参照してしまっている処理でエラーになる
- SW2.5 向けのテキスト置換定義 [>>] が正しく解決されない
